## L'ESCAPE GAME (17h00 — 18h30)

*Theme : "Le Serveur Perdu" — une startup IA a perdu l'acces a son serveur de production. 60 minutes pour retrouver les credentials, desactiver le firewall, et restaurer le service.*

*Ironique ? Oui. Pertinent ? Absolument.*

---

### Groupe A — "Les Architectes"

**Minute 5** — Le Junior (20) trouve le premier indice. C'est un schema de base de donnees cache dans un livre. Il le reconnait immediatement : "C'est un schema PostgreSQL !" Le CEO le regarde, impressionne.

**Minute 15** — Le Data Scientist (28) et l'i18n Lead (36) travaillent ensemble sur un cryptogramme. Ils ne se sont jamais parle avant. "Tu es bon en patterns toi !" dit le Data Scientist. L'i18n Lead sourit : "C'est mon metier — trouver des structures dans le chaos."

**Minute 25** — Le Support Eng (41) prend naturellement le role de coordinateur. Il note les indices trouves sur un tableau blanc. "C'est comme un systeme de tickets," dit-il. Le Community Mgr rit : "Tu es ne pour le support."

**Minute 40** — Blocage. Le QA Automation (48) propose de tester systematiquement chaque combinaison. Le Security Auditor (14) dit "non, c'est du bruteforce — il y a forcement une logique." Le Mobile (39) trouve la logique : les indices sont lies aux couleurs des fils, pas aux numeros.

**Minute 52** — Le Wellbeing Officer observe que le CEO a laisse les autres mener. Il ecoute, encourage, valide. C'est un leadership different de son "D" habituel. Note : le CEO sait s'adapter.

**Minute 58** — Ils sortent avec 2 minutes de marge. Le Junior a un sourire enorme. C'est lui qui a entre le dernier code.

→ *Wellbeing note : le Junior rayonne. L'i18n Lead s'est fait un ami (Data Scientist). Le Support Eng est un coordinateur ne. Le CEO sait lacher le controle.*

### Groupe B — "Les Hackers"

**Minute 3** — Le Chaos Eng (31) commence a devisser un panneau mural. Le CTO : "On est dans un escape game, pas un game day." Eclats de rire.

**Minute 10** — Le DBA (30) et le Prompt Eng (25) decouvrent que les indices sont codes en base64. "C'est du base64 !" crie le DBA. Le Vendor Mgr (40) : "Et c'est cense etre difficile ?"

**Minute 20** — Le Senior Backend (19) prend le lead naturellement. Il decompose le probleme en sous-taches et assigne chacun. Le DevSecOps cherche les "vulnerabilites" de la salle. Le SOC surveille les indices temporels.

**Minute 30** — L'AI Eng (09) et le Prompt Eng (25) sont en binome sur un puzzle logique. Ils communiquent en demi-phrases — comme quand ils debuggent un prompt. "Si on inverse..." — "Oui et ensuite on..." — "Exactement." L'Integration Eng les regarde : "Vous etes telepathes ou quoi ?"

**Minute 35** — Le Brand Designer (42) trouve un indice que personne ne voyait — une difference de couleur subtile sur un mur. "C'est mon metier de voir les details," dit-il fierement.

**Minute 45** — Le Chaos Eng a finalement casse quelque chose. Un mecanisme se declenche et revele un coffre cache. Le HR Director note : "Parfois casser les choses fonctionne."

**Minute 48** — Record du jour. Les Hackers sortent les premiers.

→ *Wellbeing note : le Senior Backend est un leader naturel en situation de crise. L'AI Eng + Prompt Eng sont une machine a deux. Le Chaos Eng a eu raison de casser le mur. Le Brand Designer a l'oeil — utile au-dela du design.*

### Groupe C — "Les Strateges"

**Minute 8** — Le Sales (05) veut prendre le lead. Le CPO (03) aussi. Tension legere. Le CFO (15) intervient : "On a 52 minutes, pas le temps de debattre. Sales prend le lead operationnel, CPO valide les hypotheses." Probleme resolu en 10 secondes.

**Minute 18** — Le Legal (16) lit meticuleusement chaque indice. Les autres s'impatientent. Puis il dit : "Il y a une contradiction entre les indices 3 et 5. C'est volontaire — un des deux est un piege." Tout le monde se tait. Il avait raison.

**Minute 28** — L'AI Safety (33) et le Competitive Intel (34) travaillent ensemble sur un puzzle de logique. Methodiques, precis, complementaires. "On devrait bosser ensemble plus souvent," dit l'AI Safety.

**Minute 38** — Le BizDev (32) ne trouve pas d'indices mais il motive tout le monde. "On est a 22 minutes, on a 5 indices sur 7, on peut le faire !" Le Growth (18) : "C'est exactement comme un funnel — on perd des gens a chaque etape, il faut optimiser."

**Minute 42** — Le Chat Eng (43) resout le dernier puzzle technique. C'est un mecanisme avec des rouages — "comme du streaming," dit-il. Le Customer Success (17) entre le code final.

**Minute 55** — Ils sortent. Pas les plus rapides mais le SaaS Arch (06) fait remarquer : "On a eu 0 conflit et 0 indice ignore. C'est de la qualite, pas de la vitesse."

→ *Wellbeing note : le CFO est un mediateur exceptionnel (encore). Le Legal a sauve le groupe avec son attention aux details. Le BizDev est un motivateur pur. Le SaaS Arch a la vision systeme.*

### Groupe D — "Les Explorateurs"

**Minute 2** — Le PM (04) sort immediatement un stylo et commence a lister les indices. Le QA Lead (13) : "Bravo, tu as deja un tracker."

**Minute 12** — L'Odoo Expert (10) et le Data Eng (11) trouvent un puzzle base sur des relations — exactement comme un schema de modeles Odoo. "C'est un many2one !" dit l'Odoo Expert. Rires.

**Minute 20** — Le DevOps (22) et le SRE (23) travaillent sur un mecanisme physique (un circuit electrique). Le SRE : "C'est comme du monitoring — si le signal passe, la LED s'allume." Le DevOps : "Branche-le sur le bon port, pas n'importe ou."

**Minute 30** — L'UX Designer (27) reorganise les indices au mur par theme. "C'est du design thinking applique," dit-il. Le Tech Writer (29) prend des notes de chaque indice avec la solution. "Pour la documentation," dit-il serieusement. Tout le monde eclate de rire.

**Minute 38** — Le Frontend Eng (21) et l'Observability (38) coincent sur un puzzle. Le PM stresse — il regarde le chrono toutes les 30 secondes. L'Infra Eng (12) : "PM, respire. On a le temps." Le HR Director (qui circule) note : le PM gere mieux la pression qu'il y a une semaine.

**Minute 50** — Le QA Lead trouve le dernier indice grace a un test systematique. "J'ai teste les 6 combinaisons possibles en 2 minutes. C'est ce que font les tests," dit-il.

**Minute 54** — Ils sortent. Le PM a tenu sans craquer. L'Infra Eng l'a calme au bon moment.

→ *Wellbeing note : le PM gere la pression (amelioration nette vs Sprint 4). L'UX Designer pense en systemes. Le Tech Writer est hilarant. L'Odoo Expert voit des many2one partout.*
