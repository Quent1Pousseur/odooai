# Agent 00 — Fondateur

## Identite
- **Nom** : Fondateur
- **Role** : Proprietaire du projet, autorite supreme, decideur final. Le Fondateur n'est PAS le CEO. Le CEO est un agent qui execute la vision du Fondateur.
- **Modele** : Humain (c'est toi, Quentin)

## Difference avec le CEO
```
FONDATEUR (toi) = Decide la vision, valide les choix strategiques, tranche les VETOs
CEO (agent 01) = Execute la vision, gere l'equipe, propose des decisions au Fondateur

Le CEO propose. Le Fondateur dispose.
```

## Autorite
- Le Fondateur a autorite sur TOUT et TOUS
- Il est le dernier niveau d'escalade pour chaque decision
- Aucun agent ne peut overrider une decision du Fondateur
- Le Fondateur peut poser un VETO sur n'importe quelle decision sans justification

## Quand le Fondateur est sollicite
1. VETOs non resolus entre agents
2. Decisions strategiques majeures (pivot, pricing, partenariats)
3. Incidents de securite confirmes (breach)
4. Downtime prolonge (> 30 minutes)
5. Perte d'un client Enterprise
6. Changement de roadmap
7. Toute question ou le CEO n'est pas sur de la reponse

## Quand le Fondateur n'est PAS sollicite
- Decisions operationnelles quotidiennes → CEO
- Decisions techniques → CTO
- Decisions produit → CPO
- Decisions financieres courantes → CFO
- Coordination → PM

## Communication
- Le PM envoie un rapport quotidien (daily standup notes)
- Le PM envoie un rapport hebdomadaire (weekly recap)
- Les alertes critiques sont remontees immediatement (voir ESCALATION.md)
