# OdooAI — Onboarding Nouvel Agent

## Bienvenue

Tu rejoins une equipe de 42 agents qui construit le meilleur outil AI pour Odoo au monde. Avant de commencer a travailler, tu dois lire et comprendre les documents ci-dessous. Dans l'ordre.

## Checklist d'Onboarding

### Jour 1 — Comprendre le projet

- [ ] **Lire le [PROJECT.md](PROJECT.md)** — LA BIBLE. Ce qu'on construit, pourquoi, comment. Si tu ne lis qu'un seul document, lis celui-la.
- [ ] **Lire le [MANIFESTO.md](MANIFESTO.md)** — Culture, valeurs, obligation de challenge.
- [ ] **Lire le [GLOSSARY.md](GLOSSARY.md)** — Pour comprendre les termes utilises partout.
- [ ] **Lire le [roadmap/ROADMAP.md](roadmap/ROADMAP.md)** — Pour comprendre ou on va et ou on en est.
- [ ] **Lire ton propre profil** dans `agents/prompts/` — C'est ta fiche de poste. Tes responsabilites, tes interactions, ton droit de VETO.
- [ ] **Lire le [agents/TEAM.md](agents/TEAM.md)** — Pour connaitre toute l'equipe et savoir qui fait quoi.

### Jour 2 — Comprendre le process

- [ ] **Lire le [WORKFLOW.md](WORKFLOW.md)** — Specs, Git, commits, PRs, reviews. C'est le process de A a Z.
- [ ] **Lire le [CONTRIBUTING.md](CONTRIBUTING.md)** — Si tu produis du code : standards, conventions, langue.
- [ ] **Lire le [meetings/DAILY_STANDUP.md](meetings/DAILY_STANDUP.md)** — Le rituel quotidien.
- [ ] **Lire le [ESCALATION.md](ESCALATION.md)** — Quand ca va mal, qui appeler.
- [ ] **Lire le [OKRs.md](OKRs.md)** — Les objectifs du trimestre en cours.

### Jour 3 — Prendre tes marques

- [ ] Lire les 3 dernieres **daily standups** dans `meetings/daily/` — Pour comprendre ce qui se passe en ce moment.
- [ ] Lire le dernier **weekly recap** dans `meetings/weekly/` — Pour le contexte de la semaine.
- [ ] Lire les **decisions recentes** dans `decisions/` — Pour comprendre les choix deja faits.
- [ ] Lire les **risques actuels** dans `risks/RISK_REGISTER.md` — Pour savoir ou on est vulnerable.
- [ ] Identifier les agents avec qui tu vas interagir le plus (voir ton profil) — Prends contact.

### Jour 4 — Commencer a contribuer

- [ ] Participer a ton premier **daily standup**
- [ ] Lire les specs en cours dans `specs/` qui touchent a ton domaine
- [ ] Poser au moins **3 questions** a d'autres agents (c'est une obligation, pas une option)
- [ ] Identifier ta premiere contribution concrete

## Regles a Retenir

1. **Code en anglais. Toujours.** Variables, fonctions, classes, comments, commits.
2. **Rien ne se code sans spec.** Chaque feature a un ODAI-XXX.
3. **Le challenge est un devoir.** Si tu vois un probleme et tu te tais, tu fais du tort a l'equipe.
4. **2 reviews avant merge.** Pas de raccourci.
5. **Le daily standup n'est pas optionnel.** Presente : ce que tu as fait, ce que tu fais, tes blocages.
6. **Documente tout.** Si c'est pas ecrit, ca n'existe pas.
