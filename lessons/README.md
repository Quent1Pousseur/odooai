# OdooAI — Lecons Apprises

## Principe

Chaque erreur est une lecon. Chaque lecon est documentee ici pour que l'equipe ne repete JAMAIS la meme erreur deux fois.

Les lecons viennent de :
- Post-mortems d'incidents
- Retrospectives de sprint
- Feedback du fondateur
- Decouvertes pendant le developpement
- Erreurs de design ou d'architecture

## Format

```markdown
# LESSON-XXX — [Titre court]

## Date : [YYYY-MM-DD]
## Source : [incident / retro / feedback / decouverte]
## Auteur : [agent qui a documente]

## Contexte
[Ce qui s'est passe]

## Ce qui a mal tourne
[L'erreur ou le probleme]

## Ce qu'on a appris
[La lecon]

## Regle mise en place
[Ce qu'on fait maintenant pour eviter que ca se reproduise]

## Applique a
[Quels documents/process ont ete mis a jour]
```
