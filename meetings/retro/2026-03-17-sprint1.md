## Ce qui n'a pas marche ❌

**Le plus grave du sprint.** 2 specs ecrites apres le code (DATA-002, AGENT-001). Reviews invisibles (subagent). Meetings oublies 3 fois. Le fondateur a du rappeler le processus a plusieurs reprises.

> **Fondateur** : "Je suis intransigeant la-dessus. Plus jamais."

Le pitch, les personas, le script, la liste des PME, le message type — tout est pret depuis 2 sessions. Mais zero message envoye. C'est un echec de la Piste C.

> **CEO (01)** : "C'est la responsabilite du fondateur. Les outils sont la, l'execution manque."

### 3. RDV avocat LGPL pas pris — Legal (16)
Mentionne depuis le kick-off. La question est redigee, l'analyse preliminaire est faite, mais aucun RDV avec un professionnel. Risque non-clos.

Les 9 BA Profiles sont generes mais aucun n'a ete relu par l'Odoo Expert. Ils pourraient contenir des erreurs factuelles que le LLM a hallucinees.

### 5. Tests d'integration toujours absents — QA Lead (13)
159 tests unitaires, zero test d'integration. Le OdooClient n'a jamais parle a un vrai Odoo. Le chat CLI n'a pas de tests automatises.

## Surprises 💡

### 1. Le fondateur est le meilleur QA — CPO (03)
Les bugs trouves par le fondateur (selection field crash, serve crash, system role error) sont tous des vrais edge cases que les tests unitaires n'ont pas couverts. Le test humain est irremplacable.

### 2. Les BA Profiles se generent en quelques secondes — AI Engineer (09)
On estimait $5 pour les 9 domaines. Le cout reel est probablement inferieur. Sonnet est rapide et le format JSON est bien respecte.

### 3. Le processus est aussi important que le code — Fondateur
La correction de processus (specs retroactives, reviews documentees) a pris plus de temps que prevu mais c'est un investissement necessaire. Sans tracabilite, le projet n'est pas perenne.
