## PARTIE 3 — Le VETO Legal

### Legal (16) — Presentation du probleme
Odoo utilise la licence LGPL v3. On prevoit d'extraire systematiquement le code source pour generer des Knowledge Graphs dans un produit commercial closed source.

**La question** : est-ce que cette extraction constitue une oeuvre derivee au sens de la LGPL ?

**Analyse preliminaire** :
- La LGPL autorise l'**utilisation** de bibliotheques LGPL dans un logiciel proprietary (c'est sa raison d'etre vs GPL)
- Mais nous ne **linkons** pas Odoo — nous **parsons** son code source pour en extraire des informations structurelles
- Les Knowledge Graphs sont des **faits** (noms de modeles, types de champs, contraintes) — les faits ne sont generalement pas protegeable par le copyright
- **Precedent** : les API (Oracle v Google) — la Cour Supreme US a juge que l'utilisation d'API etait fair use
- **Risque** : pas de precedent exact pour "extraction AST de code LGPL pour produit concurrent"

> **CEO (01) intervient** : "Est-ce que ca nous bloque pour Sprint 1 ?"
>
> **Legal (16)** : "Non, si on considere que les Knowledge Graphs sont des faits structurels, pas du code derive. Mais je recommande qu'on consulte un avocat specialise en open source. En attendant, on peut coder le parseur — on n'expose rien publiquement avant d'avoir l'avis."
>
> **AI Safety (33) intervient** : "Dans le meme esprit, il faut un disclaimer clair : 'OdooAI n'est pas affilie a Odoo SA.'"

**Decision du Fondateur** : On avance sur DATA-001 (le parseur est un outil interne). Je prends en charge la consultation juridique. Legal redige la question precise.
