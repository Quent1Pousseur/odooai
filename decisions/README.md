# Architecture Decision Records (ADR)

## Principe

Chaque decision architecturale importante est documentee ici. Pas pour la bureaucratie — pour que dans 6 mois, quand quelqu'un demande "pourquoi on a fait ca ?", la reponse soit ecrite.

## Format

Chaque ADR est un fichier : `ADR-XXX-titre-court.md`

```markdown
# ADR-XXX — [Titre]

## Date
[YYYY-MM-DD]

## Status
PROPOSED | ACCEPTED | DEPRECATED | SUPERSEDED by ADR-YYY

## Contexte
[Pourquoi cette decision se pose]

## Decision
[Ce qui a ete decide]

## Options Evaluees
### Option A : [nom]
- Avantages : [...]
- Inconvenients : [...]

### Option B : [nom]
- Avantages : [...]
- Inconvenients : [...]

## Justification
[Pourquoi cette option et pas les autres]

## Consequences
[Ce que ca change, ce qu'on gagne, ce qu'on perd]

## Participants
[Qui a participe a la decision]
```
